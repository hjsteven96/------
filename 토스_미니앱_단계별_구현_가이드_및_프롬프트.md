# 토스 미니앱 단계별 구현 가이드 및 AI 프롬프트

이 문서는 이 워크스페이스 내의 여러 토스 미니앱(`point-cafe`, `point-cat`, `point-pig` 등)의 소스 코드와 공식 가이드를 분석하여 작성한 **단계별 구현 가이드**와 **AI 코딩용 프롬프트 세트**입니다.

이 가이드를 활용하면 기존 미니앱의 설계적 공통점을 유지하면서 새로운 포인트 적립 미니앱을 빠르게 개발하거나, AI 어시스턴트(Cursor, Gemini 등)에게 작업을 위임해 자동으로 코드를 생성할 수 있습니다.

---

## 1. 개요 및 공통 아키텍처

워크스페이스 내의 미니앱들은 **Vite + React + TypeScript + Firebase Firestore** 기반으로 구축되어 있으며, 화면 구현에는 **TDS(Toss Design System)**를 사용합니다.

### 📂 프로젝트 폴더 구조 및 파일 역할
```text
├── granite.config.ts        # 앱인토스 메타 정보 및 로컬 포트 설정
├── firestore.rules          # Firestore 보안 규칙 (사용자별 데이터 쓰기/읽기 권한 제어)
├── firestore.indexes.json   # Firestore 인덱스 설정
├── index.html               # 뷰포트 설정 및 루트 엘리먼트 정의
└── src/
    ├── main.tsx             # Entry Point (TDS Mobile Provider 셋업, Analytics 초기화)
    ├── env.ts               # 환경 변수 로딩 및 fallback 정의 (runtimeConfig)
    ├── constants.ts         # 포인트 적립 수치, 광고 임계값, 브랜드 컬러 등 상수 정의
    ├── types.ts             # GameState, RewardType 등의 타입 선언
    ├── firebase/
    │   └── init.ts          # Firebase DB 객체 초기화 및 export
    ├── lib/
    │   ├── tossSdk.ts       # 토스 앱 브릿지 래퍼 (Mock fallback 포함)
    │   └── analytics.ts     # PostHog 분석 이벤트 로깅 래퍼
    ├── hooks/
    │   ├── useGameState.ts  # 게임 상태 관리 훅 (LocalStorage <-> Firestore 동기화)
    │   └── useLongPress.ts  # 꾹 누르기 상호작용 처리를 위한 훅
    ├── components/
    │   ├── TdsWebView.tsx   # Txt, Spacing, TdsButton 등 TDS 래핑 컴포넌트
```

### 🔄 핵심 서비스 루프 (포인트 적립 모델)
1. **유저 로그인**: Toss SDK를 통해 익명 해시 값(`userKey`) 및 토스 로그인 여부를 가져옴.
2. **상호작용 (꾹 누르기)**: 사용자가 화면 상의 메인 버튼을 꾹 누르고 있으면 게이지(완성도/친밀도 등)가 1%씩 상승.
3. **광고 허들 (Ad Unlock)**: 특정 완성도(예: 48%, 98%)에 도달하면 진행이 잠기며, 전면 또는 리워드 광고를 끝까지 시청해야 잠금이 해제됨.
4. **포인트 지급**: 임계값(예: 25%, 50%, 80%, 100%) 도달 시 '포인트 받기' 활성화. `grantTossPoints` API를 호출해 토스 포인트를 실제로 적립해주고, Firestore/로컬 상태에 기록.
5. **초기화 및 루프 반복**: 100% 보상을 획득하면 `cycleId`가 1 증가하고 완성도가 0%로 초기화되어 루프가 반복됨.

## 2. 단계별 구현 가이드

### 1단계: 초기 셋업 및 환경 설정
새 미니앱의 정체성(이름, 브랜드 컬러 등)을 정의하고 환경 변수 구조를 잡습니다.

*   **`granite.config.ts`**: 토스 플랫폼이 인식할 메타데이터와 개발 서버 포트를 선언합니다.
    ```typescript
    import { defineConfig } from "@apps-in-toss/web-framework/config";
    export default defineConfig({
      appName: "point-animal", // 콘솔/앱 식별명
      brand: {
        displayName: "포인트 동물 키우기",
        primaryColor: "#3182F6", // 대표 브랜딩 색상
        icon: "https://static.toss.im/...", // 앱 아이콘 URL
      },
      web: { host: "localhost", port: 3000 },
    });
    ```
*   **`src/env.ts`**: `import.meta.env`를 직접 참조하지 않고 `runtimeConfig` 객체로 감싸 환경 변수 누락 시의 fallback 정책을 중앙에서 통제합니다.
*   **`src/constants.ts`**: 보상 포인트액(`SMALL_REWARD_POINTS = 1`, `BIG_REWARD_POINTS = 2`), 광고 시청 임계값(`[48, 98]`), 보상 임계값(`[25, 50, 80, 100]`)을 설정합니다.

### 2단계: TDS Mobile 적용 및 UI 화면 구현
토스 앱 내부 웹뷰처럼 자연스러운 프리미엄 UI를 위해 TDS(Toss Design System)를 연동합니다.

> [!IMPORTANT]
> **TDS CSS 및 Provider 설정 누락 주의**
> - `@toss/tds-colors/colors.css`(기본 토큰)와 `@toss/tds-colors/colors.light.css`(adaptive 색상 변수)가 `main.tsx`에 반드시 포함되어야 합니다.
> - 이 설정이 없으면 바텀시트 배경이 투명하게 변하는 렌더링 버그가 발생합니다.

*   **`src/main.tsx`**:
    ```tsx
    import { TDSMobileProvider } from "@toss/tds-mobile";
    import "@toss/tds-colors/colors.css";
    import "@toss/tds-colors/colors.light.css";

    const browserUserAgent = {
      fontA11y: undefined,
      fontScale: 1,
      isAndroid: false,
      isIOS: true,
      colorPreference: "light" as const,
      safeAreaBottomTransparency: "opaque" as const,
    };

    ReactDOM.createRoot(document.getElementById("root")!).render(
      <TDSMobileProvider userAgent={browserUserAgent}>
        <App />
      </TDSMobileProvider>
    );
    ```
*   **`src/components/TdsWebView.tsx`**: 원본 TDS 컴포넌트를 사용하기보다 프로젝트 테마에 맞게 래핑하여 디자인 일관성을 높입니다.
    - `Txt`: 타이포그래피(title, body 등)와 굵기(weight) 래핑
    - `TdsButton`: 브랜드 컬러가 적용된 버튼 컴포넌트
    - `Spacing`: 수직 간격(margin 대신 사용 권장)
    - `PointRewardOverlay`: 포인트를 획득했을 때 터지는 축하 팝업 오버레이

### 3단계: 로컬 및 Firebase Firestore 상태 관리
미니앱은 비로그인 브라우저 개발 환경과 실제 토스앱 인증 환경을 모두 커버하기 위해 **LocalStorage**와 **Firebase Firestore** 간 양방향 동기화 모델을 사용합니다.

*   **동기화 전략 (`useGameState.ts`)**:
    - **초기 로드 (Hydration)**: 유저가 로그인하면 LocalStorage 상태와 Firestore 상태를 불러와 `cycleId`가 더 높거나, 완성도가 더 높은 쪽의 데이터로 병합(`mergeLoadedState`)합니다.
    - **상태 업데이트 & Debouncing**: 게이지가 증가할 때마다 LocalStorage에는 **동기적(즉시)**으로 저장하고, Firestore 서버에는 불필요한 네트워크 요청을 줄이기 위해 **500ms 디바운스(Debounce)**를 적용하여 저장합니다.
*   **인프라 배포**:
    - Firestore 데이터 보호를 위해 `firestore.rules`를 적용하고, 배포합니다.
    ```javascript
    rules_version = '2';
    service cloud.firestore {
      match /databases/{database}/documents {
        match /users/{userId} {
          allow read, write: if request.auth != null && request.auth.uid == userId;
        }
      }
    }
    ```

### 4단계: 토스 SDK 연동 및 포인트 적립
`src/lib/tossSdk.ts`는 `@apps-in-toss/web-framework` 인터페이스를 래핑하여 토스 앱 내부와 일반 브라우저(Mock) 대응을 일원화합니다.

*   **로그인 및 식별키 획득 (`getUserKey`)**:
    - 일반 브라우저: `crypto.randomUUID()`를 통해 고유 ID를 생성해 LocalStorage에 캐싱.
    - 토스 앱: `getAnonymousKey()` 브릿지를 호출하여 사용자 해시 값을 획득.
*   **포인트 적립 (`grantTossPoints`)**:
    ```typescript
    import { grantPromotionReward } from "@apps-in-toss/web-framework";

    export async function grantTossPoints(promotionCode: string, amount: number) {
      if (!isTossApp()) return { success: true }; // 브라우저 테스트 Mocking
      
      const result = await grantPromotionReward({
        params: { promotionCode, amount }
      });
      // 성공/실패 여부 분기 후 리턴
    }
    ```
*   **광고 잠금 해제 (`presentFullScreenAd`)**:
    - `loadFullScreenAd`를 통해 광고를 백그라운드에서 미리 로드(`prewarmFullScreenAd`)해둡니다.
    - 잠금 구간에 도달하면 `showFullScreenAd`로 전면/보상형 광고를 띄우고, `userEarnedReward` 또는 `dismissed` 이벤트를 받아 완수 여부를 확인 후 게이지 잠금을 해제(`completeAdUnlock`)합니다.

---

### 5단계: QA 및 부가 기능 연동
*   **햅틱 피드백 (`triggerTickHaptic`)**: 사용자가 버튼을 꾹 누르는 매 초마다 `generateHapticFeedback({ type: "tickWeak" })`을 실행하여 물리적 반응을 제공합니다.
*   **물리 뒤로가기 제어 (`registerBackEventHandler`)**: 토스 앱 내에서 물리 뒤로가기 버튼을 누를 때 샌드박스가 바로 닫히지 않고 경고창이나 오버레이를 닫도록 `useBackEvent` 또는 브릿지 이벤트를 바인딩합니다.
*   **공유 및 딥링크**: `getTossShareLink` API와 `share` API를 조합하여 사용자에게 전송할 공유 카드를 생성합니다.
